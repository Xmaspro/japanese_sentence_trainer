# Handover: Japanese Sentence Trainer Checkpoint

Date: 2026-05-31

## Current Stable Goal

The app is now basically usable as a visual-novel style Japanese dialogue trainer. The current checkpoint focuses on keeping the five daily-life themes stable, with AI roleplay available as an optional speaking practice layer.

## How To Run

```powershell
cd D:\code\japanese_sentence_trainer
npm start
```

Open:

```text
http://127.0.0.1:5177/japanese_sentence_trainer/index.html
```

If port `5177` is already in use, stop the old Node process first:

```powershell
Get-NetTCPConnection -LocalPort 5177 | ForEach-Object { Stop-Process -Id $_.OwningProcess }
```

## Current Feature State

- Main UI keeps the five dialogue themes:
  - 日常生活
  - 学校学习
  - 旅行出行
  - 医院健康
  - 社交娱乐
- Training keeps listening dictation and replacement/free expression.
- Dialogue context is shown as message-style bubbles.
- Progress is counted by completed dialogue groups.
- Microsoft TTS, VOICEVOX TTS, browser speech fallback, Gemini explanations, temporary recording/playback, and AI roleplay are available.

## Recent Fixes In This Checkpoint

- Fixed AI scene setup modal overlap with subtitles.
- Fixed the "use current course scene" flow so the AI setup modal can be interacted with cleanly.
- Added in-memory speech Blob caching and AI speech prefetching to reduce repeat TTS latency.
- Added a stricter AI roleplay gate:
  - Gemini must return explicit `accepted: true` before the roleplay continues.
  - If Gemini gives correction or expression-improvement feedback, the frontend blocks progression even if the model tries to continue.
  - Rejected user turns remain visible for review but are excluded from the next Gemini history payload.
